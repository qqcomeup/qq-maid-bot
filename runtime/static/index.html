网页技术架构

当前轻量方案

  • 前端：单文件 HTML + 原生 JavaScript
  • 托管：qq-maid-bot 内部 LLM HTTP Axum 静态服务 /console
  • API：对外保留 /healthz、/console/ 和 /api/v1/markdown/render
  • Markdown：页面调用 /api/v1/markdown/render 由 Rust 后端解析；生产发送仍走 Rust Gateway MarkdownPayload
  • 后续：可把更多 QQ Markdown 约束和 rig.rs 侧输出规范化集中到同一个 Rust 模块
