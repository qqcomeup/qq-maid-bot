# 任务：将 Gateway 和 LLM 合并为一个可执行程序

当前项目需要分别启动：

- `qq-maid-gateway-rs`
- `qq-maid-llm`

请将它们合并为一个统一可执行程序，用户启动一次即可同时运行 Gateway 和 LLM。

## 要求

1. 先阅读仓库根目录的 `AGENTS.md`、`README.md`、workspace `Cargo.toml`，并检查两个项目现有的 `main.rs`。
2. 把两个 `main.rs` 中的实际启动逻辑提取为可复用函数。
3. 新增一个统一入口，同时启动：
   - LLM HTTP 服务
   - QQ Gateway
   - 现有后台任务
4. 暂时保留现有调用链：
   - Gateway 继续通过本机 HTTP 调用 LLM
   - 本次不要改成进程内直接调用
5. 确保 LLM 完成监听后，再正式启动 Gateway，避免刚启动时请求失败。
6. 日志、配置、`.env` 等全局初始化只能执行一次。
7. `Ctrl+C` 或 SIGTERM 时，两个服务都应正常退出。
8. 任一核心服务异常退出时，整个程序应返回错误，不要留下半运行状态。
9. 尽量兼容现有环境变量、端口、接口和部署方式。
10. 不要修改 Todo、RSS、天气、聊天等业务逻辑。
11. 不要进行无关重构。

## 旧入口

先检查 GitHub Actions、Docker、systemd、README 和脚本是否依赖原来的两个二进制。

可以暂时保留旧入口，但必须复用新的启动函数，不要保留两套重复逻辑。

## 验收标准

- 构建后有一个推荐使用的统一可执行文件。
- 只启动该程序，Gateway 和 LLM 都能正常运行。
- Gateway 仍能正常调用 `/v1/respond`。
- 原有机器人功能不受影响。
- `Ctrl+C` 后进程能完整退出。
- 日志和配置不会重复初始化。
- 原有测试、构建和 Clippy 检查通过。
